Design Notes

1. Summary Bar (Status Overview)

Status Chip: Reading/Connected/Disconnected/Error count
Click to filter by status
50 readers' current state at a glance

2. Three View Modes

Grid: Default card view with name, status, indicators, IP
Compact: Minimal view showing only name + status color
List: Table view with all details in rows

3. Card Design (Top Status Bar)

Status color on top edge (not left side) for better scanability
Name + 3 indicators (top row)
Status badge + IP (bottom row)
Hover: tooltip with full details + shadow elevation

4. Color System

Disconnected (Gray)
Connected (Green)
Reading (Blue, pulse)
Error (Red)

5. Indicator System

Beep (Green circle)
Warning Light (Orange circle)
Buzzer (Blue circle)
Disabled (Gray square outline)

6. Interaction

Hover: card elevation + tooltip
Right-click: context menu (connect/disconnect/inventory/antenna)
Search: real-time filter by name or IP
Status chip click: filter by status

Layout Calculation (Grid View)

  • Card width: 108px (min) ~ responsive (auto-fill)
  • 900px window: ~8 cards/row, 50 readers = ~7 rows
  • 1200px window: ~10 cards/row, 50 readers = 5 rows
  • 1600px window: ~13 cards/row, 50 readers = 4 rows

Layout Calculation (Compact View)

  • Card width: 72px (min) ~ responsive
  • 900px window: ~12 cards/row, 50 readers = ~5 rows
  • 1200px window: ~16 cards/row, 50 readers = ~4 rows

Swing Implementation Notes

  • Status bar: top edge (3px height) instead of left side for better visual scanning
  • CSS grid auto-fill equivalent: WrapFlowLayout (already implemented)
  • Hover tooltip: JToolTip with HTML or custom JPopupMenu
  • Summary bar: JPanel with FlowLayout + custom painted chips
  • View toggle: CardLayout switching between grid/list panels
  • Search filter: DocumentListener on JTextField, hide/show cards
