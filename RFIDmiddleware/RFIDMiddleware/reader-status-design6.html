Design #6 Key Concepts

Hexagonal Honeycomb

50개 리더기를 벌집 구조 육각형으로 배치. 육각형은 사각형 대비 15% 더 높은 공간 효율, 인접 리더기와의 시각적 연결감 제공. 상태별 색상이 벌집 패턴과 결합하여 유기적이고 생동감 있는 대시보드 구성.

3-Column Layout

좌측 통계 사이드바(상태 분포 + 필터) + 중앙 허니콤 그리드 + 우측 상세 패널. 정보의 계층이 명확: 전체 현황(좌) → 개별 위치 확인(중) → 상세 정보(우). 클릭 흐름이 자연스러움.

Interactive Detail Panel

육각형 클릭 시 우측 패널에 확대 육각형 + 연결정보 + 인디케이터 상태 + 최근 활동 로그 표시. 호버 툴팁과 상세 패널의 이중 레벨 정보 접근.

Dual View Mode

Hex(육각형 벌집) / Grid(컴팩트 직사각형) 전환. 육각형은 시각적 임팩트, 그리드는 밀도와 실용성. 상태별 필터 클릭으로 특정 상태만 하이라이트.

Swing Implementation

  • Hexagon rendering: Custom JPanel, paintComponent에서 Polygon(6 vertices) 그려 clip path 없이도 구현 가능. GeneralPath로 정육각형 좌표 계산.
  • Honeycomb layout: Custom LayoutManager, 짝수/홀수 행 오프셋 계산. hexWidth=60, rowHeight=52 (높이의 75%), 홀수행은 halfWidth만큼 오프셋.
  • Glow effect: Reading 상태 헥사곤에 AlphaComposite.SrcOver로 반투명 확대 헥사곤 그리기 + Timer로 opacity 애니메이션.
  • 3-column layout: BorderLayout(WEST=stats, CENTER=honeycomb, EAST=detail). 스크롤은 JScrollPane.
  • Click interaction: MouseListener on hex → detailPanel 업데이트. paintComponent에서 Polygon.contains(mousePoint)로 클릭 감지.
  • Status filter: stat-card 클릭 시 필터 상태 저장, repaint에서 비대상 hex에 AlphaComposite(0.15f) 적용.
