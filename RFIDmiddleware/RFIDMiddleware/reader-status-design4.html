Design #4 Key Concepts

Swim Lane Layout

Each status = one collapsible lane. Readers flow into their status lane as colored pills. Instantly answers "which readers need attention?" by looking at the Error lane. Lane size reflects reader count visually.

Stacked Bar (Top)

Proportional distribution of all 50 readers in a single horizontal bar. Each segment is clickable to scroll to that lane. Shows exact counts. Status at a glance in 0.3 seconds.

Reader Pills

Compact colored chips: status dot + name + 3 indicator dots. Hover for full tooltip (IP, port, RSSI, indicator details). Colored background + border matches status. Minimal footprint, maximum density.

Collapsible Lanes

Click lane header to collapse/expand. Hide offline readers to focus on active ones. Progress bar in header shows proportion. Collapse Disconnected lane (27 readers) to save space.

Swing Implementation

  • Stacked bar: Custom JPanel, paintComponent draws colored fillRect segments proportionally.
  • Swim lanes: Vertical BoxLayout of lane panels. Each lane = JPanel(BorderLayout) with header + WrapFlowLayout body.
  • Collapsible: Toggle body panel visibility on header click, revalidate parent.
  • Reader pills: Custom painted JPanel with rounded rect background, inline dot + text + indicator dots.
  • Lane reflow: When reader status changes, remove pill from old lane, add to new lane, revalidate both.
  • Search filter: DocumentListener on JTextField, setVisible(false) on non-matching pills.
